arXiv Vanity used to render academic papers from arXiv as responsive web pages so you didn't have to squint at a PDF.

arXiv now has HTML papers, so arXiv Vanity doesn't need to exist any longer.

arXiv Vanity was built by Ben and Andreas. When we were on vacation in Greece, Andreas was squinting at an arXiv paper on his phone. "Why the hell are these papers PDFs," he complained, and we discovered our mutual hatred for PDF academic papers.

"Why aren't they on the Web?" we wondered. The Web was invented at CERN (a physics institution!) for sharing academic writing. We have <figure> tags, <author> tags, <cite> tags. Hyperlinks are for citations. But, 30 years later, the Web has Facebook and academic papers are still PDFs. That's not what it was designed for!

Instead of having a vacation, we built the first version of arXiv Vanity. arXiv is kinda neat in that they don't just have the PDF â€“ they keep the original LaTeX source. PDF is just presentational, whereas LaTeX has a lot of semantic information in it, which makes it tractible to convert to HTML.

We originally used Pandoc, but TeX is turing complete and Pandoc tried to naively parse it, so that didn't really work.

Later we came across LaTeXML, a brilliant LaTeX runtime written in Perl. Weirdly it is a project of the US Government, created by Bruce Miller at NIST. The NIST Digital Library of Mathematical Functions is a library of government approved mathemetical formulae, represented in the gold standard of math rendering: LaTeX. LaTeX isn't a particularly good archival medium though. It doesn't really have a specification and is only defined by a runtime that Don Knuth wrote in the 70s in his own custom programming language (called "WEB", funnily). So, NIST created a runtime from scratch in Perl to convert the LaTeX into semantic XML. It ballooned from its original intention of converting LaTeX formulae to XML, adding support for more LaTeX features, more output formats like HTML, eventually becoming one of the best LaTeX runtimes.

Our friend Mark Hurrell made some beautiful designs for the papers, then we slapped some CSS and JavaScript on top of the LaTeXML output and wrapped it all up in a tool called Engrafo. Then, we made a little single use app called arXiv Vanity that rendered arXiv papers as HTML on-demand. People seemed to like it.

It caught arXiv's attention and they wanted to make it official. We got involved in early projects to add HTML to arXiv but we got busy and didn't contribute much in the end.

Thankfully Deyan Ginev and team from LaTeXML and ar5iv made it happen and finally HTML is on the abstract page. They've been working on this for years. We know how hard it is to get LaTeX rendering well, and to get it this good is no small feat.

Our job here is done. Down with PDFs! Web is best!
